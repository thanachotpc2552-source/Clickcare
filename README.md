# 🚀 Hackathon Challenge: The "Living" App (48 Hours)

ยินดีต้อนรับสู่สนามประลองไอเดียและฝีมือโค้ด! ภารกิจของคุณคือ **"ชุบชีวิตข้อมูล JSON ที่นิ่งสนิท ให้กลายเป็น Web Application ที่ใช้งานได้จริง มีชีวิตชีวา และแก้ปัญหาได้ตรงจุด"**

---

## 🎯 ภารกิจของคุณ (The Mission)
คุณได้รับ **"Multi-Vertical Data Pack"** ซึ่งประกอบด้วยข้อมูลจำลองจาก 9 อุตสาหกรรม (เช่น ร้านค้า, โรงแรม, การแพทย์, การเงิน ฯลฯ)

**สิ่งที่คุณต้องทำ:**
1.  **เลือก 1 Vertical** (หรือจะผสมผสานหลาย Vertical เข้าด้วยกันก็ได้)
2.  **พัฒนา Mini App (MVP)** ที่เปลี่ยนข้อมูลดิบเหล่านั้นให้เป็นหน้าระบบที่สวยงามและใช้งานง่าย
3.  **ใส่ฟีเจอร์เด็ด** เพื่อแสดงศักยภาพของข้อมูลออกมาให้มากที่สุด

---

## 🛠️ ข้อกำหนดขั้นต่ำ (Requirements)
*ต้องมีครบทุกข้อจึงจะผ่านเกณฑ์การส่งงาน*

### 1. หน้าสรุปข้อมูล (Dashboard / Overview)
* แสดงข้อมูลภาพรวมที่สำคัญ (KPIs) เช่น ยอดขายรวม, จำนวนงานที่ค้าง, หรือรายรับรายจ่าย
* ต้องมีการนำเสนอด้วย **กราฟ หรือ แผนภูมิ** อย่างน้อย 1 จุด

### 2. รายการและค้นหา (List & Search)
* แสดงข้อมูลเป็นตาราง (Table) หรือการ์ด (Card Grid)
* มีระบบ **ค้นหา (Search)** และ **กรองข้อมูล (Filter)** (เช่น กรองตามสถานะ, วันที่, หมวดหมู่)

### 3. หน้ารายละเอียด (Detail Page)
* เมื่อคลิกที่รายการ ต้องเข้าไปดูรายละเอียดเชิงลึกได้ (เช่น กดที่ Order เพื่อดูรายการสินค้าข้างใน)

### 4. คุณภาพทางเทคนิค (Tech & Quality)
* **Responsive Design:** แสดงผลได้ดีทั้งบน Mobile และ Desktop
* **Performance:** คะแนน Lighthouse (หมวด Performance และ Accessibility) **ต้อง ≥ 90**

---

## ✨ โบนัสพิเศษ (Bonus Tracks)
*ทำเพิ่มเพื่อชิงคะแนนพิเศษ +10 คะแนน*

* **🤖 AI Integration:** นำ AI มาช่วยวิเคราะห์ข้อมูล, แนะนำสินค้า, หรือทำ Chatbot (แนะนำให้ใช้ Gemini Free Tier)
* **⚡ Real-time Simulation:** จำลองเหตุการณ์ข้อมูลไหลเข้า (เช่น มี Order ใหม่เด้งขึ้นมาโดยไม่ต้องกด Refresh)
* **📊 Deep Insight:** วิเคราะห์ข้อมูลเชิงลึกหา Trend หรือ Pattern ที่ซ่อนอยู่
* **📱 PWA / Offline-first:** แอปใช้งานได้ลื่นไหลแม้เน็ตหลุด

---

## 📂 โครงสร้างข้อมูล (Data Structure)
ข้อมูลทั้งหมดอยู่ในโฟลเดอร์ `data/` โดยมีไฟล์หลักคือ:
* `users.json`: ข้อมูลผู้ใช้งาน (ใช้เชื่อมโยงกับ `user_id` ในไฟล์อื่นๆ)
* โฟลเดอร์ย่อยตามหมวดหมู่: เช่น `ecommerce`, `finance`, `healthcare` ฯลฯ
* *โปรดอ่านความหมายของตัวแปรและ Status Code ในไฟล์ `DATA_DICTIONARY.md`*

---

## 🤖 คำแนะนำเรื่อง AI (AI Usage)
หากคุณต้องการใช้ AI ในโปรเจกต์ (เช่น Gemini, ChatGPT)
* **Bring Your Own Key:** ให้ผู้เข้าแข่งขันสมัคร API Key แบบ Free Tier ด้วยตนเอง
* **Recommendation:** แนะนำ **Google Gemini API** (Free Tier) เนื่องจากฟรี, รวดเร็ว และรองรับ Context ได้ยาว (เหมาะกับการโยน JSON เข้าไปประมวลผล)
